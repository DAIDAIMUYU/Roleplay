# 第二阶段执行计划

## 使用方式

- `[ ]` 未开始
- `[~]` 进行中
- `[x]` 已完成
- 每完成一项，就在本文件打勾。
- 每个任务尽量独立提交，避免把多个阶段混在一个 commit。

## 推荐执行顺序

1. A 阶段：先做真实设备和设置页稳定性验收。
2. B 阶段：修移动端和设置页体验。
3. C 阶段：增强聊天与上下文。
4. D/E 阶段：扩展 Provider 与数据同步。
5. F/G 阶段：推进 App 化评估和技术债拆分。

## 当前不建议做

- 不建议立即全量 App 化。
- 不建议一次性接所有 Provider。
- 不建议继续扩大 PWA 页面内安装提示逻辑。
- 不建议大改 Context Builder 核心。
- 不建议在没有验收设计前改数据库 schema。
