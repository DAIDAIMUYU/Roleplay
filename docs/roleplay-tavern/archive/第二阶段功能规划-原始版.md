# 第二阶段功能规划

## 1. 当前已完成能力概览

- 角色卡：支持创建、编辑、软删除、角色选择与聊天绑定。
- 聊天房间：支持会话、消息分页、消息编辑、版本切换、重新生成、上下文控制台。
- 本地优先数据：未登录可使用浏览器本地数据库，登录后保留本地镜像。
- 云端同步：支持本地/云端手动同步入口与数据管理说明。
- 托管加密凭据：支持 hosted_encrypted API Key，前端不展示明文 Key。
- API 配置编辑：支持 Provider 预设、模型选择、测试、启用、编辑和删除。
- 用量与费用：支持通用 usage 展示；DeepSeek 支持缓存命中、费用估算和余额查询入口。
- DeepSeek 缓存诊断：支持 recent context runs 展示、缓存健康提示和压缩建议。
- 手动压缩上下文：已有摘要/压缩入口，长会话可使用摘要降低上下文压力。
- PWA 基础能力：保留 manifest、Service Worker、更新提示和离线提示；下线页面内安装提示卡片。
- 帮助中心：已重构为普通用户指南，覆盖快速开始、API、安全、本地/云端和移动端排查。

## 2. 必须优先修复的稳定性问题

| 优先级 | 事项 | 原因 | 建议处理 |
| --- | --- | --- | --- |
| P0 | PWA 旧缓存更新策略验证 | 移动端可能继续运行旧 JS，影响用户判断修复是否生效 | 用真实 iOS Safari、Android Chrome、PWA standalone 做更新提示和清缓存回归 |
| P0 | 移动端真实设备回归 | Modal、底部导航、输入框聚焦和安全区需要实机确认 | 建立固定验收清单，逐页录屏验证 |
| P0 | 设置页 API 配置最终验收 | hosted / local / session 三类配置容易混淆 | 确认账号云端配置、本设备配置、启用状态、编辑和删除都符合预期 |
| P1 | 文案编码清理 | 历史产品记录中存在乱码段落，影响维护 | 不影响运行，建议单独做文档清理，不混入功能开发 |
| P1 | hosted-provider-balance 部署核对 | DeepSeek 余额查询依赖 Edge Function 部署 | 部署后验证 hosted_encrypted 余额查询 |

## 3. 体验优化类

- 移动端顶部栏进一步压缩，减少聊天页首屏占用。
- 设置页卡片操作菜单简化，把编辑、测试、启用、删除在移动端整理成更清楚的两行或更多菜单。
- 帮助中心搜索，便于快速定位“API 失败”“缓存命中”“数据丢失”等问题。
- 首页动态预览优化，让首次打开更直观看到“配置 API → 创建角色 → 聊天”的路径。
- 深色模式与主题切换，作为 P2 外观增强，不影响主流程。
- 空状态文案继续收口，避免“可以直接聊天”的误导。

## 4. 聊天与上下文增强

- 缓存趋势条优化：长会话下历史点位可能过密，需要聚合或滚动查看。
- recent context runs 内存缓存：减少频繁切换会话时的上下文诊断读取。
- sendMessage 后上下文估算补齐：发送后立即更新上下文窗口估算，减少等待下一次刷新。
- 长会话自动摘要提示：上下文使用率超过 70% 时提示用户考虑压缩摘要。
- 摘要编辑体验增强：用 AppModal 替代 `window.confirm` 或过渡式确认。
- 手动压缩历史版本管理：允许查看、回滚或对比摘要版本。
- 角色版本历史：当前消息版本已实现，角色卡自身版本历史可单独规划。
- 快捷指令：`/summary`、`/memory`、`/scene`、`/ooc` 可作为后续聊天效率增强。

## 5. API / Provider 扩展

- 非 DeepSeek Provider 价格估算：OpenAI、OpenRouter、SiliconFlow、Moonshot、Qwen 等价格表需要独立维护，暂不硬接。
- 更多 OpenAI Compatible 模型支持：继续扩展预设模型列表，但保持“测试不等于启用”的体验。
- Provider 级测试详情：显示 HTTP 状态、模型可用性、余额/额度提示，但不能泄露 Key。
- 余额查询扩展：当前仅 DeepSeek，后续可按 Provider 能力逐个接入。
- 错误诊断面板：把 401、403、429、5xx、网络失败翻译成更易懂的用户提示。
- Gemini / Claude 专用 adapter：当前预设标记为需要专用适配器，可通过 OpenRouter 中转，后续再实现原生 adapter。

## 6. 数据与同步

- 云端同步冲突解决 UI：同 id 内容不同不应静默覆盖，需要展示差异和保守合并策略。
- 数据导入导出增强：增加更严格的 schema 校验和导入前风险提示。
- 备份版本管理：支持本地备份历史、备注和恢复点。
- 多设备同步状态显示：显示最后同步时间、差异数量、冲突数量和当前设备镜像状态。
- 回收站彻底删除：如实现必须二次确认，并清楚说明不可恢复。

## 7. 移动端 / PWA / App 化

- PNG PWA 图标兼容：补齐不同尺寸图标，减少浏览器安装警告。
- PWA 更新体验验证：确保新版可见、可刷新、不会长期卡旧代码。
- 安卓 WebView / Capacitor 包装：评估本地文件系统、通知、系统分享能力。
- iOS App 可行性评估：重点关注本地数据库、后台限制、审核风险和 API Key 安全策略。
- 手机表单体验增强：长表单分段、底部保存按钮常驻、键盘弹出后的可视区域测试。

## 8. 开发侧技术债

- `ChatRoomPage.tsx` 拆分：移动端头部、空状态、角色选择、新建会话弹窗可拆为子组件。
- `ContextPreview.tsx` 拆分：状态、角色、模板、世界书、记忆、摘要、用量与费用、高级信息分区可拆开。
- Settings API 模块拆分：云端账号配置、本设备配置、编辑弹窗、测试状态分别独立。
- Studio Editor 表单抽象：角色、模板、世界书、记忆编辑器复用字段、footer、safe-area 和错误处理。
- 自动化回归测试：覆盖移动端新建会话、创建角色、设置页 API 编辑、Modal 遮挡。
- Playwright E2E：至少覆盖 `/`、`/roleplay`、`/studio`、`/settings`、`/help` 的基础冒烟。

## 9. 推荐优先级

### P0：影响真实使用

1. 移动端真实设备回归。
2. PWA 更新与缓存策略验证。
3. 设置页 API 配置最终验收。
4. hosted_encrypted 余额查询部署与验证。

### P1：明显体验优化

1. 帮助中心搜索。
2. 设置页移动端操作菜单简化。
3. 长会话自动摘要提示。
4. ContextPreview 分区拆分。
5. 同步冲突解决 UI。

### P2：后续增强

1. 深色模式和主题切换。
2. 非 DeepSeek 价格估算。
3. Gemini / Claude 原生 adapter。
4. PWA/App 化。
5. 快捷指令和角色版本历史。

## 10. 下一步建议

1. 先稳定移动端和设置页，确保真实设备上新建会话、创建角色、API 编辑、Modal、底部导航都可靠。
2. 再补缓存诊断细节，重点是 sendMessage 后估算、趋势条和摘要提示。
3. 再扩展 Provider，逐个接入价格、余额和专用 adapter，不要一次性接完。
4. 最后考虑 App 化，先验证 PWA 更新和移动端表单体验，再评估 Capacitor / iOS App。
