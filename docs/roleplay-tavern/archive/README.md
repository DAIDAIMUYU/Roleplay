# 开发历史归档

本目录保存的是项目开发过程中的历史记录，**不是当前产品的主文档**。

如果想了解产品现在的真实状态，请查阅上层目录的主文档。

## 目录说明

- `产品界面优化与体验重构记录.md` — 阶段 13 之后的 UI、移动端、PWA、文档整理历史记录
- `Provider支持矩阵.md` — Provider 支持矩阵历史版，当前说明见上层 `03-API与Provider说明.md`
- `第二阶段功能规划-原始版.md` — 第二阶段规划原始素材，当前执行清单见上层 `06-第二阶段执行计划.md`
- `legacy-planning/` — 早期产品、架构、阶段路线、测试策略和开发提示词
- `legacy-guides/` — 旧版用户指南、部署指南、安全说明、架构说明、开发者指南等
- `development-history/phase-records/` — 阶段 0-11 执行记录
- `development-history/phase-records/progress/` — 每周/每阶段进度记录
- `development-history/audits/` — 阶段审计报告和验收清单
- `development-history/screenshots-or-notes/` — 过程截图和零散笔记
- `development-history/需求追加与修改记录.md` — 阶段 11 需求变更追踪

## 注意事项

- 这些文档中的"Demo"、"阶段 N"、"Mock AI"等措辞反映了开发过程中的历史概念，不代表当前产品状态。
- 当前产品已从"登录/云端优先"重构为"本地优先，可选云端同步"。
- 如果你是新用户或新开发者，建议先查看主目录下的产品文档。
- 归档文件不建议直接作为开发任务来源；后续执行请优先看 `../06-第二阶段执行计划.md`。
