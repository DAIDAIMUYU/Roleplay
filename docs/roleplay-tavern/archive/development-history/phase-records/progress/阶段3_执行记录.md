## 三、Provider Gateway 说明

```
providerGateway.ts
  ├── getAvailableProviders() → ProviderMeta[]
  │     └── 从 AVAILABLE_PROVIDERS 读取（mock / deepseek / openai_compatible）
  ├── getProviderMeta(id) → ProviderMeta
  ├── resolveRuntimeMode(isGuest, storageMode) → ProviderRuntimeMode
  │     ├── demo_mock     : Guest/Demo — 强制 Mock
  │     ├── byok_session  : 已登录 + session_only 保存
  │     ├── byok_local_device : 已登录 + local_device 保存
  │     └── hosted_encrypted_future : 阶段 8，当前不实现
  ├── buildConfig(overrides) → ModelProviderConfig
  ├── buildConfigFromStorage(provider, key, mode) → ModelProviderConfig
  ├── validateConfig(config) → AppProblem | null
  ├── testProviderConnection(config) → TestResult
  └── sendProviderRequest(isGuest, config, messages) → ChatResult
```

**核心安全规则**：
- Guest/Demo → 永远 Mock，不可绕过
- Authenticated → 依据选中的 Provider + API Key 调用真实 API
- 无 API Key → 不可调用真实 Provider

## 四、Mock Provider 说明

- **用途**：Guest / Demo 模式专用
- **行为**：随机返回 4 条中文模拟回复之一，附带用户输入回显
- **标识**：所有回复以 `[Demo Mock]` 开头，明确可识别
- **网络**：不发起任何 HTTP 请求
- **延迟**：模拟 300-700ms 响应（提升 Demo UX）
- **Token**：inputTokens 和 outputTokens 均返回 0
- **数据**：不写入正式数据库

## 五、BYOK 本地保存说明

### Level 1：session_only（仅本次会话）
- 存储位置：内存 Map（页面关闭即丢失）
- 实现：`saveApiKeySession()` / `loadApiKey()` / `clearApiKey()`
- 适用：临时体验、测试 API

### Level 2：local_device（本设备保存）
- 存储位置：`localStorage`（`rp_tavern_` 前缀）
- 实现：`saveApiKeyLocalDevice()` / `loadApiKey()` / `clearApiKey()`
- 安全提示：写入时 console.warn 明确告知"不上传服务器"
- UI 提示：设置页面明确显示"当前 API Key 仅保存在本设备浏览器中"

### Level 3：hosted_encrypted_future
- **阶段 3 不实现**
- 设置页面以禁用态展示（灰色 + "后续阶段 8 实现"标签）
- 类型系统已预留 `hosted_encrypted_future`

## 六、API Key 安全边界

| 操作 | 允许 | 禁止 |
|---|---|---|
| 保存到内存 | ✅ session_only | — |
| 保存到 localStorage | ✅ local_device（带警告） | — |
| 保存到 Supabase | — | ❌ 阶段 3 禁止 |
| 保存到数据库 | — | ❌ 任何阶段禁止明文 |
| 写入日志 | — | ❌ 禁止 |
| 写入错误消息 | — | ❌ 禁止 |
| 写入 context_runs | — | ❌ 禁止 |
| Admin 查看 Key | — | ❌ 禁止 |

## 七、DeepSeek Provider 支持情况

- **连接测试**：`GET /models` 验证 API Key 和网络连通性
- **基础 chat**：`POST /chat/completions` 非流式调用
- **默认 baseURL**：`https://api.deepseek.com/v1`
- **默认 model**：`deepseek-chat`
- **错误**：401/402/429/5xx/网络错误 全部翻译为中文
- **流式输出**：阶段 4 实现

## 八、OpenAI-compatible Provider 支持情况

- **连接测试**：`GET /models`（需要自定义 baseURL）
- **基础 chat**：`POST /chat/completions`
- **必须提供**：baseURL + model + apiKey
- **错误翻译**：与 DeepSeek 共享同一套翻译逻辑
- **通用性**：任何 OpenAI API 兼容的服务商均可使用

## 九、Settings 页面交互说明

API Provider 设置卡片提供完整交互：

1. **Provider 下拉选择**：Mock / DeepSeek / OpenAI Compatible
2. **Model 输入**：默认填充 Provider 默认模型
3. **Base URL 输入**：默认填充 Provider 默认地址（显示/隐藏切换）
4. **API Key 输入**：密码框模式，可切换显示
5. **保存模式 3 选 1**：
   - 仅本次会话（启用）
   - 保存在当前设备（启用）
   - 加密托管到账号（禁用 + "后续阶段 8 实现"）
6. **操作按钮**：保存配置 / 测试连接 / 清除
7. **连接结果**：成功（绿色卡片 + 模型列表）/ 失败（红色卡片 + 中文错误信息）
8. **安全横幅**：amber 色提示"Key 仅保存在本设备"
