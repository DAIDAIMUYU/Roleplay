## 三、角色卡管理说明

### 存储策略
- 使用已有 `characters` 表，结构化字段全部存入 `card_json` (jsonb)
- 不新增列、不新增表

### 字段映射
| UI 字段 | 存储位置 | 类型 |
|---|---|---|
| 名称 | `characters.name` | text |
| Emoji | `characters.avatar_emoji` | text |
| 标签 | `characters.tags` | text[] |
| 收藏 | `characters.is_favorite` | boolean |
| 身份/外貌/性格/背景/说话风格/关系/阶段/称呼/开场白/禁止规则/喜欢/扩展 | `characters.card_json` | jsonb |

### 操作
- **创建**：CharacterEditor 抽屉 → `Repository.createCharacter()`
- **编辑**：CharacterEditor 抽屉（预填数据）→ `Repository.updateCharacter()`
- **删除**：软删除 → `Repository.deleteCharacter()` (设置 deleted_at)
- **归档**：`Repository.archiveCharacter()` (设置 archived_at)
- **收藏切换**：`Repository.updateCharacter({ is_favorite })`
- **搜索**：按 name/summary/card_json.identity 前端过滤
- **标签筛选**：前端 tags 数组匹配

## 四、提示词模板说明

### 存储
- 使用已有 `prompt_templates` 表
- `category`：general/daily/plot/narration/style/rules/group

### 模板变量
支持 6 个变量（模板编辑器中可点击插入）：
- `{{character_name}}` — 角色名称
- `{{user_name}}` — 用户称呼
- `{{current_scene}}` — 当前场景
- `{{relationship_stage}}` — 关系阶段
- `{{speaking_style}}` — 说话风格
- `{{greeting}}` — 开场白

### 操作
- **创建/编辑**：TemplateEditor 抽屉
- **删除**：`Repository.deletePromptTemplate()`
- **收藏**：切换 `is_favorite`
- **搜索**：按 title/description
- **分类筛选**：category 芯片
- **标签筛选**：tags 芯片

## 五、会话绑定角色说明

### 数据库
- 使用已有 `sessions.primary_character_id` 字段
- 不新增表

### 流程
1. 用户点击「新会话」→ 弹出角色选择器 Modal
2. 选择角色 → `createSession(characterId)` → 写入 `primary_character_id`
3. 如果角色有 greeting → 自动创建第一条 assistant 消息并保存
4. 选择已有会话 → `selectSession()` → 加载绑定的角色
5. 会话列表显示角色 emoji + 名称
6. 顶部栏显示当前角色信息

## 六、角色提示词注入说明

### 时机
每次 `sendMessage()` 时自动构建

### 优先级
1. 如果有 `activeTemplate`（提示词模板）→ 变量替换后作为 system 消息
2. 如果没有模板 → 从 card_json 构建默认 System Prompt
3. 如果也没有角色绑定 → 不发送 system 消息（普通聊天）

### 注入内容
默认 System Prompt 包含：角色名称、身份、性格、外貌、背景、说话风格、关系、阶段、称呼、喜欢、禁止规则、角色保持指令

### 效果
有了角色提示词注入后，AI 回复会体现角色设定，而不是普通助手。这是阶段 5 的核心交付。

## 八、Demo / User 数据边界

- **Demo 模式**：Studio 页显示"登录后使用"空状态，Chat 使用内存 mock 数据，不写数据库
