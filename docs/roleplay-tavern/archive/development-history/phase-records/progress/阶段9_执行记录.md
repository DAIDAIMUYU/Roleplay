## 消息分页实现方式

- Repository 新增 `listMessagesPage(sessionId, branchId, beforeCreatedAt/beforeId, limit)`。
- 主线消息分页只返回：
  - `deleted_at is null`
  - `superseded_by_message_id is null`
  - `hidden = false`
  - 当前 `branch_id`
- 初始进入会话时仅加载最近 50 条消息。
- 顶部“加载更早消息”继续向前查询更旧消息。
- 聊天页在 prepend 旧消息前记录滚动高度，加载完成后恢复相对位置，避免跳到底部。
- 发送新消息与流式回复仍会自动滚动到底部。

## Context Builder 如何取消息

- `buildCurrentContext()` 在未传 `messagesOverride` 时，会单独从 DB 拉取最近 40 条有效消息。
- 这样即使 UI 只加载了一页消息，Provider 实际上下文仍能拿到最近必要消息。
- 删除消息、被编辑替换的旧 assistant、非当前分支消息都不会进入 `providerMessages`。

## AI 记忆建议实现方式

- 用户点击“AI 提炼记忆”后，才调用当前 Provider。
- 会话级提炼默认读取最近 20 条有效消息；单条提炼则只基于该条消息。
- 先要求模型返回 JSON 数组，字段包括：
  - `memory_type`
  - `title`
  - `content`
  - `salience`
  - `reason`
- 前端先解析并展示预览，不直接写入数据库。
- 用户确认后：
  - “保存并启用” -> 写入 `memories.status = active`
  - “保存到收件箱” -> 写入 `memories.status = suggested`

## suggested / active / disabled / deleted 规则

- `suggested`：候选记忆，待确认，不注入上下文。
- `active`：正式记忆，可被当前会话绑定并进入上下文。
- `disabled`：已忽略 / 已关闭，不注入上下文。
- `deleted`：已删除，不注入上下文。

## 单条消息提炼记忆实现方式

- 在消息气泡 hover 操作区新增“提炼记忆”按钮。
- 用户消息和 AI 消息都可触发。
- 点击后弹出同一套候选记忆预览弹窗。
- 仅改变预览与后续保存，不会直接写 active 记忆。

## 安全边界说明

- API Key 不进入记忆建议草稿。
- API Key 不进入 `context_runs`。
- API Key 不进入导出 JSON。
- API Key 不进入 console 日志与用户可见错误信息。
- `hosted_encrypted` 模式下，记忆建议仍通过既有 Edge Function 代理链路调用 Provider，前端不接触托管明文 Key。
