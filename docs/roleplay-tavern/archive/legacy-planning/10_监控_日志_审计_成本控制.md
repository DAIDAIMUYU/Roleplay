# 10_监控_日志_审计_成本控制

## 日志字段

每次聊天请求至少记录：

- request_id
- context_run_id
- user_id
- mode
- provider
- model
- session_id
- branch_id
- input_tokens
- output_tokens
- cache_hit_tokens
- latency_ms
- cost_usd
- error_code
- retryable

## 脱敏规则

- 不记录完整 API Key。
- 不记录 service role key。
- 默认不记录完整 prompt。
- IP 与 UA 使用 hash/截断。
- 管理员默认看聚合数据，不看用户原始私聊。

## 审计事件

记录：

- 登录/登出
- Provider 测试
- 导出备份
- 导入备份
- 删除/恢复
- Admin 角色变更
- break-glass
- 修改 Demo 数据
- 修改系统模板

## 成本控制

产品层直接展示：

- 本轮 input tokens
- 本轮 output tokens
- 本轮预估成本
- 当前上下文长度
- 世界书注入条数
- 记忆注入条数
- 是否触发缓存命中

策略：

- Demo 不调用真实 Provider
- BYOK 用户消耗自己的 API
- Owner API 只对 Owner 可用
- Provider Gateway 做每用户/每会话/每 IP 限流
- 429 指数退避
- 402 余额不足提示

## 环境隔离

- dev
- preview
- production

各环境使用独立 Supabase 项目或至少独立 schema/配置，不混用生产数据。
