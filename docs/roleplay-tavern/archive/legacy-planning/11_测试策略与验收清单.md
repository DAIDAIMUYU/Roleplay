# 11_测试策略与验收清单

## 测试目标

1. 防止 AI/开发者乱改导致回归。
2. 防止真实模型调用浪费钱。
3. 防止 RLS 越权。
4. 防止移动端核心聊天不可用。

## 测试层级

| 层级 | 工具 | 范围 |
|---|---|---|
| 单元测试 | Vitest | contextBuilder、budgeter、provider error mapping |
| 组件测试 | React Testing Library | 表单、聊天组件、状态组件 |
| E2E | Playwright | Demo、登录、BYOK、聊天、导入导出 |
| 安全扫描 | ZAP Baseline | headers、公开端点、基本漏洞 |
| RLS 测试 | SQL / integration | 用户 A/B 数据隔离 |

## 必测路径

- 访客进入 Demo，不登录、不写库、不调真实 Provider。
- 用户注册/登录成功。
- 用户未配置 Provider 时不能真实聊天。
- 用户配置临时 Key 后测试连接。
- 创建角色、创建会话、发送消息。
- 编辑用户消息后重发。
- AI 回复失败后重试。
- 上下文预览显示纳入项与裁剪项。
- 用户 A 不能读取用户 B 数据。
- 导出备份成功。
- 误删会话可恢复。

## 阶段门禁

每阶段必须通过：

- lint
- typecheck
- unit tests
- 至少一条关键 E2E
- 文档更新
- 变更总结
