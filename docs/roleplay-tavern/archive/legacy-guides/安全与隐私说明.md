# 安全与隐私说明

## 你的数据保存在哪里

### 本地模式（未登录）

所有数据保存在当前浏览器的 **IndexedDB** 中：
- 角色卡、提示词模板、世界书与词条、记忆
- 会话记录、消息、消息版本历史
- API 配置元数据（不包含 API Key，API Key 单独存储）

> IndexedDB 是浏览器提供的本地数据库。刷新网页、关闭浏览器、重启设备后通常仍会保留。但清除浏览器网站数据、更换浏览器、更换设备、使用无痕模式或格式化设备后数据会永久丢失。

### 云端同步模式（已登录）

登录后，数据会保存到 Supabase 云端数据库中。当前设备同时保留一份本地镜像（IndexedDB），用于加速访问。

## API Key 三种保存方式

| 模式 | 保存位置 | 持久性 | 安全性 | 多设备 |
|------|---------|-------|--------|--------|
| 仅本次会话 | 浏览器内存 | 刷新后清除 | 较高（不存在磁盘） | 不支持 |
| 当前设备 | 浏览器 localStorage | 长期保存 | 存在 XSS 风险 | 不支持 |
| 托管加密 | Supabase Edge Function | 长期保存 | 较高（前端无明文） | 支持 |

### local_device 风险说明

`local_device` 模式将 API Key 明文保存在浏览器 localStorage 中。这意味着：

- 如果页面存在 XSS（跨站脚本）漏洞，攻击者可能读取你的 API Key
- 如果你使用的浏览器扩展有恶意行为，可能读取 localStorage
- 在共享设备上，其他使用同一浏览器的人可能读取 localStorage

**我们不建议在不信任的设备上使用 local_device 模式。** 如果你需要在多设备间安全使用 API Key，建议使用托管加密模式（需登录）。

### hosted_encrypted 说明

托管加密模式将 API Key 通过 Supabase Edge Function 使用 AES-256-GCM 加密后存储。前端不会保存或展示明文 Key。聊天时通过 Edge Function 代理调用 AI API。

当前限制：托管加密模式暂时不支持流式（streaming）响应。

## 什么不会被导出

导出 JSON 备份**不会**包含：
- API Key 明文
- 托管加密凭据的加密密钥
- encrypted_api_key
- encryption_iv
- provider_credential_secrets

## 什么不会被同步

本地↔云端同步**不会**传输：
- API Key 明文
- 托管加密凭据的 secret
- local_device 模式下保存的 API Key

同步只涉及角色、模板、世界书、记忆、会话、消息等业务数据。

## 你应该怎么做来保护数据

1. **定期导出备份**：在数据管理页面导出 JSON 备份，保存到安全位置。
2. **使用安全的 API Key 模式**：如果担心 XSS 风险，使用 `session_only` 或 `hosted_encrypted`。
3. **不要在他人设备上保存 API Key**：使用 `session_only` 模式可以避免 Key 被保存在浏览器存储中。
4. **不要在公开代码中提交 API Key**：`.env` 文件中的 Key 不要提交到 Git 仓库。
5. **备份不是同步**：云端同步帮助多设备访问，但不能替代定期备份。导出 JSON 备份是最可靠的恢复方式。

## 如何删除数据

- 本地数据：清除浏览器网站数据（在浏览器设置中操作）会删除所有本地 IndexedDB 数据。
- 云端数据：在数据管理页面可以软删除，或在 Supabase 控制台直接管理。
- API Key：在设置页可以清除本地 API 配置或删除托管凭据。
