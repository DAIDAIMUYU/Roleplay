# 开发者指南

## 技术栈

| 层 | 技术 |
|---|---|
| 框架 | Vite 5 + React 18 + TypeScript 5 |
| 样式 | Tailwind CSS 3 |
| 路由 | React Router 6 |
| 后端服务 | Supabase (Auth + Database + RLS + Edge Functions) |
| 本地存储 | IndexedDB |
| AI 调用 | 多 Provider Gateway (DeepSeek / OpenAI-compatible) |

## 项目结构

```
src/
  app/            App Shell + Router
  pages/          路由页面 (Landing, ChatRoom, Studio, Settings, Login, DataManagement, Admin, Demo)
  features/
    auth/         Supabase Auth + AuthProvider
    roleplay/
      providers/      Provider Gateway + Presets
      hooks/          核心 Hooks (useChatSession, useCharacters, etc.)
      components/     UI 组件 (chat/, settings/, studio/)
      repositories/   数据访问层
      services/       业务服务 (sync, dataManagement, hostedCredentials)
      storage/        本地存储 (apiKey, apiProviderConfig)
      context/        Context Builder
      types/          类型定义
      utils/          工具函数
  shared/
    components/     通用 UI 组件
    hooks/          通用 Hooks
    styles/         全局样式
supabase/
  migrations/       SQL 迁移文件
  functions/        Edge Functions
docs/              项目文档
```

## 如何启动开发环境

```bash
npm install
cp .env.example .env
# 编辑 .env 填入 VITE_SUPABASE_URL 和 VITE_SUPABASE_ANON_KEY
npm run dev
```

本地模式不需要 Supabase 也能运行大部分功能。

## 数据访问层规则

### Repository 模式

**组件不能直接调用 Supabase**。所有数据库访问必须经过 Repository 层：

```typescript
// 正确：通过 Repository
import * as Repo from "../repositories/roleplayRepository";
const chars = await Repo.listActiveCharacters(supabase, userId);

// 错误：直接调 Supabase
const { data } = await supabase.from("characters").select("*");
```

### 本地/云端切换

Hook 层通过 `isLocalMode` 自动判断使用 LocalRepo 还是 CloudRepo：

```typescript
const rows = isLocalMode
  ? await LocalRepo.listActiveCharacters()
  : await Repo.listActiveCharacters(supabase, userId);
```

### 本地镜像

登录用户在云端写入成功后，LocalMirror 自动将数据镜像到本地 IndexedDB。镜像写入失败不阻塞主流程。

## 不要做的事情

- 不要把 API Key 写入业务数据表
- 不要把 API Key 写入 context_runs
- 不要把 API Key 打印到 console
- 不要在组件中直接调用 Supabase
- 不要在前端硬编码 API Key 或 Secret

## 如何新增 Provider

1. 在 `providerPresets.ts` 中添加新的 ProviderPreset 定义
2. 在 `provider.types.ts` 的 `ProviderType` 中添加新 ID
3. 如果是 OpenAI-compatible 接口，设置 `compatibility: "openai_compatible"`
4. 如果是新的专用 API，需要在 `providers/` 中新增 ProviderAdapter 实现
5. 在 `providerGateway.ts` 的 `adapters` Map 中注册新 adapter

## 如何新增数据表

1. 在 `supabase/migrations/` 中创建新的 SQL 迁移文件
2. 在 `src/features/roleplay/types/database.ts` 中新增 Row 类型
3. 在 `roleplayRepository.ts` 和 `localRoleplayRepository.ts` 中新增 CRUD 方法
4. 如需本地镜像，在 `localMirror.ts` 中新增 mirror 函数
5. 如需同步，在 `localCloudSyncService.ts` 中新增上传/下载逻辑
