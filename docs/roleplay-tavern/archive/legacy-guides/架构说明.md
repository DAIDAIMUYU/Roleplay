# 架构说明

## 总体架构

Roleplay Tavern 采用 **Local-first + 可选云端同步**的架构：

```
┌─────────────────────────────────────────────┐
│                  前端 (React SPA)              │
│                                              │
│  ┌──────────┐  ┌──────────┐  ┌───────────┐  │
│  │  UI 页面  │  │  Hooks   │  │ Providers │  │
│  └────┬─────┘  └────┬─────┘  └─────┬─────┘  │
│       │             │              │         │
│  ┌────┴─────────────┴──────────────┴──────┐  │
│  │         Repository Gateway              │  │
│  │   (LocalRepo / CloudRepo / LocalMirror) │  │
│  └────┬────────────────────────┬──────────┘  │
│       │                        │             │
│  ┌────┴──────┐          ┌─────┴──────┐      │
│  │ IndexedDB  │          │  Supabase  │      │
│  │ (本地数据)  │          │ (云端数据)  │      │
│  └───────────┘          └────────────┘      │
└─────────────────────────────────────────────┘
```

## 核心概念

### Local-first 设计

未登录用户拥有完整的本地使用体验。所有数据通过 IndexedDB 持久化，刷新后数据保留。

登录是可选的。登录后获得云端同步和多设备互通能力，但不会丢失本地功能。

### Repository 模式

组件不直接访问 IndexedDB 或 Supabase。所有数据操作通过 Repository 层统一管理：

- **LocalRepo** (`localRoleplayRepository.ts`)：IndexedDB 本地仓库
- **CloudRepo** (`roleplayRepository.ts`)：Supabase 云端仓库
- **LocalMirror** (`localMirror.ts`)：云端写入后自动镜像到本地

### 同步中心

- **localCloudSyncService**：管理本地↔云端双向同步
- 上传：读取本地数据 → 写入 Supabase（保留 ID、user_id 重设、冲突副本策略）
- 下载：读取云端数据 → 写入 IndexedDB（不覆盖本地不同内容）
- 冲突策略：保守处理，不静默覆盖，不同内容生成副本

### Provider Gateway

- **providerPresets**：Provider 预设定义（Base URL、模型列表、兼容性）
- **providerGateway**：统一调用入口，根据 preset 映射到实际 adapter
- **适配器**：deepseekProvider / openAICompatibleProvider / mockProvider

### API Key 管理

- **session_only**：浏览器内存（Map），刷新清除
- **local_device**：浏览器 localStorage，明文保存
- **hosted_encrypted**：Supabase Edge Function AES-256-GCM 加密存储，前端无明文

### 数据安全边界

```
业务数据 (IndexedDB / Supabase)
  ├── 角色、模板、世界书、记忆、会话、消息
  └── 不含 API Key、不含凭据 secret

API Key 存储 (单独管理)
  ├── session_only → 内存
  ├── local_device → localStorage
  └── hosted_encrypted → Supabase Edge Function

同步/导出 不传输
  ├── API Key 明文
  ├── encrypted_api_key
  ├── encryption_iv
  └── provider_credential_secrets
```
