### 首页快速开始区说明

1. **配置 API** — 选择 DeepSeek、OpenAI-compatible 或其他 Provider，填入 API Key。未配置 API 时仍可使用本地预览模式。

3. **开始聊天** — 进入聊天室，选择角色并创建会话后即可开始角色扮演。

# 阶段 14.1：登录态 hosted_encrypted 支持流式输出 + 统一 DeepSeek streaming usage 采集

- **问题**

  - 登录账号后云端托管模式（hosted_encrypted）用量无法正常返回
  - 托管模式缺少流式支持，前端收到的是整段非流式回复
  - 本地和托管走两套不同逻辑，usage 采集不一致
- **修复目标**
  - hosted_encrypted 支持流式输出（SSE text/event-stream）
  - DeepSeek 本地模式和 hosted_encrypted 模式统一采集 streaming usage
  - 前端统一一套流式消费逻辑
  - 不泄露 API Key
  - 不破坏 session_only / local_device / hosted_encrypted 三种模式
- **修改文件清单**
  - `supabase/functions/_shared/provider.ts` — 新增 `providerChatStream()`，通过 ReadableStream 向前端转发 DeepSeek SSE，emit delta/usage/done/error 事件
  - `supabase/functions/hosted-provider-chat/index.ts` — 支持 `stream: true`，流式路径调用 `providerChatStream` 返回 `text/event-stream`，非流式路径保持向后兼容
  - `src/features/roleplay/services/hostedCredentialsService.ts` — 新增 `sendHostedProviderChatStream()`，通过 fetch + ReadableStream 消费 Edge Function SSE，转换为回调（onDelta/onUsage/onDone/onError）
  - `src/features/roleplay/providers/providerGateway.ts` — `sendProviderStreamRequest` 中 hosted_encrypted 分支改为流式实现（80ms 轮询积攒 delta chunks 并逐条 yield），不再走非流式 `sendHostedProviderChat`
  - `src/pages/ChatRoomPage.tsx` — 两处 badge "托管·非流式" / "托管模式 · 非流式" 改为 "托管·流式" / "托管模式 · 流式"
  - `src/features/roleplay/hooks/useChatSession.ts` — 更新托管用量缺失时的降级提示文案
- **详细说明**
  - Edge Function 不再将完整响应返回为 JSON，而是在 `stream:true` 时返回 SSE 事件流
  - SSE 事件格式：`event: delta\ndata: {"text":"..."}\n\n` / `event: usage` / `event: done` / `event: error`
  - 前端 `sendHostedProviderChatStream` 通过 `fetch` 直连 Edge Function URL，读取 `response.body.getReader()` 并解析 SSE
  - providerGateway 使用轮询式 async generator（80ms 间隔），将回调积攒的 delta 逐条 yield 给 `useChatSession`
  - `useChatSession` 无需修改，仍通过 `sendProviderStreamRequest` 的 `AsyncIterable<ChatStreamChunk>` 统一消费
  - DeepSeek `stream_options: { include_usage: true }` 在 Edge Function 层转发，usage chunk 在流中捕获并标准化
  - 流式被中止（AbortController）后正确取消 Reader，不会 hang
- **完成状态**
  - 已完成（前端+Edge Function）
- **需要部署的 Supabase Functions**
  - `hosted-provider-chat` — 必须重新部署才能支持流式
- **遗留问题**
  - 轮询式 delta yield（80ms）不如事件驱动实时，但避免了 providerGateway 内嵌套 Promise 的复杂度
  - 非 DeepSeek 的 hosted_encrypted 流式（如 OpenAI Compatible）暂时不包含 stream_options（后续可扩展）
