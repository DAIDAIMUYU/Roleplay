# 配置与部署

## 1. 本地开发启动

```bash
npm install
npm run dev
```

本地模式可以在缺少 Supabase 配置时体验部分功能。云端同步和托管加密凭据需要 Supabase。

## 2. 常用检查命令

```bash
npx tsc -b
npm run build
```

## 3. 环境变量说明

只写变量名和用途，不要提交真实值。

| 变量 | 用途 |
| --- | --- |
| `VITE_SUPABASE_URL` | 前端连接 Supabase 项目 |
| `VITE_SUPABASE_ANON_KEY` | Supabase anon key |
| `API_KEY_ENCRYPTION_SECRET` | Edge Function 加密托管 API Key 的服务端密钥 |

生产环境请使用平台 Secrets，不要把真实 secret 写入仓库。

## 4. Supabase 配置

需要执行项目已有 migrations，并启用 Auth、RLS 和 Edge Functions。不要关闭 RLS。

## 5. Edge Functions 部署

按使用场景部署：

```bash
npx supabase functions deploy hosted-provider-chat
npx supabase functions deploy hosted-provider-balance
npx supabase functions deploy hosted-credentials-save
npx supabase functions deploy hosted-credentials-list
npx supabase functions deploy hosted-credentials-test
npx supabase functions deploy hosted-credentials-delete
```

- 托管加密聊天需要 `hosted-provider-chat`。
- DeepSeek 托管余额查询需要 `hosted-provider-balance`。
- 保存、列表、测试、删除托管凭据需要对应 `hosted-credentials-*` functions。

## 6. Vercel 部署

1. 连接 GitHub 仓库。
2. 设置前端环境变量。
3. 执行默认构建命令：`npm run build`。
4. 配置 SPA fallback。
5. 部署后验证首页、聊天、工坊、设置、帮助中心。

## 7. PWA 注意事项

- PWA manifest 和 Service Worker 保留。
- 页面内“安装角色酒馆”提示卡片已下线。
- 用户仍可通过浏览器菜单或分享按钮添加到主屏幕。
- 如果看到旧页面，清理站点数据或重新添加 PWA。

## 8. 部署后验收

- `npx tsc -b` 通过。
- `npm run build` 通过。
- 首页 GitHub 链接正确。
- 设置页 API 配置可保存、编辑、测试、启用。
- 移动端底部导航不遮挡 Modal。
- 托管加密凭据不返回明文 Key。
