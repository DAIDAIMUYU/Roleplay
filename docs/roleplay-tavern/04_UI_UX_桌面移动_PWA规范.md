# 04_UI_UX_桌面移动_PWA规范

## 设计原则

1. 聊天页服务聊天，不服务全部管理。
2. Studio 服务设定管理，不抢聊天主视觉。
3. 移动端按 App 设计，不压缩桌面三栏。
4. 访客必须清楚知道自己处在 Demo Mode。
5. 登录用户必须清楚知道当前使用哪个 Provider、哪个模型、谁的 API Key。
6. 参考图只作为风格方向，不要求像素级复刻。

## 视觉方向

- 背景：浅灰白 / 浅紫白 / 深色主题后期可选
- 主色：紫色、靛蓝、轻微霓虹感
- 卡片：圆角 16-24px、轻阴影、半透明可选
- 按钮：胶囊按钮、明确主次
- 消息：用户右侧气泡、角色左侧卡片
- 字体：正文舒适、长文本可读，不追求炫技

## 桌面端布局

```text
顶部：Logo / Hall / Studio / Chat / Settings / 用户状态 / 成本摘要
左栏：角色、会话、标签、搜索、新建入口
中栏：聊天消息流、流式输出、输入框、消息操作
右栏：上下文控制台、世界书命中、记忆注入、token/cost
底部：Provider / Model / 模式 / 网络状态 / 自动保存
```

## 移动端布局

底部 Tab 建议：

```text
大厅 | 聊天 | 工坊 | 设置
```

移动端 Chat：

- 顶部：返回、角色头像、会话名、更多
- 中间：消息流
- 底部：输入框、工具按钮、发送/停止
- 高级面板：使用 Bottom Sheet，不用复杂弹窗
- 键盘适配：优先 VisualViewport，VirtualKeyboard 作为渐进增强

## 必须设计的状态

- Demo Mode
- 未登录
- 已登录但未配置 Provider
- API Key 测试成功
- API Key 无效
- Provider 限流
- Provider 余额不足
- 无角色
- 无会话
- 无世界书
- 无记忆
- 正在生成
- 停止生成
- 生成失败
- 上下文超限
- 导入成功/失败
- 备份成功/失败
- 网络断开

## Onboarding

用户登录后不应直接进入空页面，应进入引导：

1. 连接 Provider / 粘贴临时 API Key
2. 测试连接
3. 创建或导入角色
4. 创建第一条会话
5. 开始聊天

允许跳过，但必须显示“未配置 Provider 时只能使用 Demo/Mock”。

## PWA 策略

PWA 是增强，不是主路径。阶段 8 前至少完成：

- Manifest
- App icon
- display-mode 适配
- 移动端 safe-area
- 基础安装引导

后期再做：

- App Shortcuts
- 离线查看历史
- 离线草稿
- 推送通知
