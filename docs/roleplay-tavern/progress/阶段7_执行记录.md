- `context_runs.message_id` 继续指向最新 assistant 消息。
- 会话删除链路保持软删除，不影响历史数据结构。

## 实现方式

### 1. 用户消息版本切换

- 编辑用户消息时：
  - 旧内容写入 `message_revisions`
  - 当前 `messages.content_text` 更新为新内容
  - `revision_no` 递增
- 首屏只批量查询 revision count，不一次性拉取所有 revision 文本。
- 点击版本箭头时再懒加载该消息的 revision 详情。

### 2. assistant 回复版本切换

- 编辑用户消息并重发后：
  - 旧 assistant 回复不回到主线
  - 新 assistant 回复保存为新的 `messages` 记录
  - 旧 assistant 回复内容写入新 assistant message 对应的 `message_revisions`
- UI 里同样可以对 assistant 消息左右切换查看旧/新版本。

### 3. 主线消息规则

- 旧 assistant 回复保留 `deleted_at`
- `superseded_by_message_id` 指向替代它的新消息
- 主线只显示当前有效消息，不显示被替换历史版本

## 安全与兼容

- 版本切换只影响 UI 展示，不修改主线消息，不重新生成，不额外写库。
- 不影响：
  - 世界书命中
  - 记忆注入
  - 摘要注入
  - Token 预算
  - Context Builder
  - 数据导出导入
