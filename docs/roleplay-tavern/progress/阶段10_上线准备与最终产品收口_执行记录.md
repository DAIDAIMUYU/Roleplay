## 4. 移动端交互补齐说明

**问题**: 消息气泡操作按钮（复制、编辑、删除、重生成、版本切换、提炼记忆）依赖 `group-hover:opacity-100`，移动端无 hover 事件，按钮不可见。

**方案**:
- 桌面端（md+）：保持原有 hover 交互，`hidden md:flex` 显示操作栏。
- 移动端（<md）：
  - 版本切换箭头常驻显示（`flex md:hidden`，无 opacity 过渡）。
  - 新增 `⋯` 按钮（MoreHorizontal 图标），点击弹出 popover 菜单。
  - Popover 菜单包含：版本切换、复制、编辑（仅用户消息）、重新生成（仅 AI 消息）、提炼记忆、删除。
  - 点击菜单外部自动关闭。
- 版本切换在移动端 popover 内和气泡上均可操作。
- 菜单使用 `absolute` 定位在消息气泡下方，不遮挡输入框。

## 5. hosted 非流式提示说明

**问题**: 托管加密模式通过 Edge Function 代理调用，当前为非流式回复，用户可能误以为卡顿。

**方案**:
- Settings 页面：info 横幅文案更新为"托管模式当前通过 Edge Function 代理请求，暂不支持逐字流式输出，回复会在生成完成后一次性显示。"
- ChatRoom 页面：桌面端顶部栏和移动端顶部栏均显示 `托管模式 · 非流式` 浅蓝色标签。
- 提示使用 sky 色系（非红色），属于信息提醒而非错误警告。
- session_only / local_device 模式不显示此提示。

## 6. 体积优化说明

**优化前**: 主 JS 包 619.95 kB (gzip: 168.24 kB)，超过 500 kB chunk warning。

**优化方案**: 路由级 code splitting
- 使用 `React.lazy(() => import(...).then(m => ({ default: m.Component })))` 模式
- 9 个页面全部拆分为独立 chunk
- AppShell 保持在首屏
- `Suspense` fallback 显示"页面加载中..."

**优化后**:
| Chunk | 大小 | Gzip |
|-------|------|------|
| index (主入口) | 399.01 kB | 114.25 kB |
| ChatRoomPage | 81.84 kB | 22.82 kB |
| StudioPage | 45.70 kB | 10.71 kB |
| DataManagementPage | 31.29 kB | 8.50 kB |
| SettingsPage | 17.18 kB | 5.47 kB |
| 其他页面 | < 8 kB each | — |

主包下降 35.6%，chunk warning 消除。

**注意事项**: SettingsPage 中 `buildConfigFromStorage` 和 `testProviderConnection` 改为直接从 `providerGateway.ts` 导入，避免 barrel export 导致的循环依赖警告。
