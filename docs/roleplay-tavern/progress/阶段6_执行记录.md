- **作用范围**：worldbook 级别使用现有 scope 字段

---

## 四、世界书条目触发说明

**触发引擎** (`worldbookTrigger.ts`)：

1. **输入来源**：当前用户消息 + 最近 6 条消息 + 当前角色名称
2. **匹配方式**：`input.toLowerCase().includes(trigger.toLowerCase())`
3. **过滤条件**：
   - `entry.enabled === true`
   - `entry.scope` 匹配（global 总是生效，character 检查当前角色，session 预留）
4. **排序**：按 `priority DESC`
5. **预算裁剪**：Token 预算超限时跳过低优先级条目
6. **结果**：`{ triggered: TriggeredEntry[], skipped: SkippedEntry[] }`

不做向量检索。不做 embedding。

## 五、记忆系统说明

- **存储**：`memories` 表（memory_type, salience, status, session_id, character_id）
- **类型**：user_preference / relationship / event / long_term / short_term / summary / character_preference
- **状态**：active（注入） / disabled（不注入） / suggested（预留） / deleted
- **注入**：只有 status='active' 的记忆注入，按 salience DESC 排序
- **手动管理**：创建/编辑/删除/启用禁用
- **AI 建议**：阶段 6 不做，留到阶段 9

## 七、Context Builder 结构

```
contextBuilder.buildContext(input) → ContextBuildOutput:
  ├── systemPrompt (完整系统提示词)
  ├── providerMessages (发送给 Provider 的消息数组)
  ├── budget (Token 预算分配)
  ├── triggerResult (世界书触发结果)
  ├── debugInfo (构建统计)
  └── estimatedTokens (估算 Token 数量)
```

## 十、Token 预算说明

- 默认总预算：**8000 tokens**
- 分配：角色+模板 ≤2000 / 世界书 ≤2000 / 记忆 ≤1500 / 摘要 ≤800 / 消息 = 剩余
- 估算：`Math.ceil(text.length / 1.5)`（中文保守估算）
- 裁剪：世界书按 priority DESC，记忆按 salience DESC，超预算截断
