## 导出备份说明

- 导出内容：
  `characters`、`character_revisions`、`prompt_templates`、`worldbooks`、`worldbook_entries`、`memories`、`sessions`、`branches`、`session_participants`、`messages`、`message_revisions`
- `context_runs` 默认不导出，用户可手动勾选高级选项。
- 导出文件名：
  `roleplay-tavern-backup-YYYYMMDD-HHmm.json`
- 导出元信息：
  `schema_version`、`app_name`、`exported_at`、`export_type`、`counts`、`data`
- 安全边界：
  不读取、不导出、不记录 API Key。

## 导入恢复说明

- 先选文件，再解析校验，再展示预览，不会在选中文件后立刻写库。
- 默认导入模式为“作为副本导入”，会为导入数据生成全新 id。
- 可选“跳过已存在项目”，按顶层名称/标题近似匹配，并跳过对应子树。
- 会保持这些外键关系：
  `sessions -> branches`
  `session_participants -> sessions / characters`
  `messages -> sessions / branches / characters`
  `message_revisions -> messages`
  `worldbook_entries -> worldbooks`
  `character_revisions -> characters`
- 导入失败会停止流程，并尽力回滚本次新写入数据，避免半残数据静默残留。

## 回收站说明

- 当前回收站直接读取各业务表的 `deleted_at`，因为现有删除链路并没有统一写入 `trash_items`。
- 支持筛选：
  会话、消息、角色、提示词模板、世界书、世界书条目、记忆
- 支持恢复：
  - 会话
  - 消息
  - 角色
  - 模板
  - 世界书
  - 世界书条目
  - 记忆
- 恢复世界书时，支持“只恢复本体”或“恢复并恢复条目”。

## 安全边界说明

- API Key 不导出。
- API Key 不进入 backup JSON。
- API Key 不进入本阶段新增的备份元数据记录。
- 导入时不会接受外部 `user_id` 作为当前身份，所有导入数据都会重写归属为当前登录用户。
- 普通用户只能操作自己在 RLS 下可见的数据。
- 本阶段没有关闭 RLS。

## 已知未完成项

- `trash_items` 仍未接入现有删除流，本阶段没有强行把它变成唯一事实来源。
- 回收站“彻底删除”按钮本阶段未实现，避免引入不可恢复删除风险。
- “跳过已存在项目”是近似匹配策略，不做危险的同 id 覆盖。

## 测试方法

1. 进入 `/settings/data`。
2. 直接导出 JSON，检查文件名和 `counts`。
3. 验证导出内容不包含 API Key。
4. 重新选择导出的 JSON，确认先出现预览，再点导入。
5. 使用“作为副本导入”，确认原数据不被覆盖，新会话可打开，新消息可显示。
6. 删除会话、世界书、消息后进入回收站，确认能看到对应记录。
7. 恢复会话/世界书后返回主流程验证状态已刷新。
