# 阶段 8.1 API 密钥服务端加密托管执行记录

**执行时间**: 2026-05-19  
**阶段目标**: 在保留阶段 3 本地 BYOK 的前提下，新增 hosted_encrypted 托管加密模式，实现跨设备同步、安全存储与 Edge Function 代理调用。

## 达成目标

- 正式实现 `hosted_encrypted` 托管模式，不再停留在 `hosted_encrypted_future` 预留状态。
- 新增双表结构：`provider_credentials` 保存可展示元数据，`provider_credential_secrets` 保存密文和 IV。
- 新增 Edge Functions 完成托管凭据的保存、列出、测试、设为默认、删除与聊天代理。
- 前端 Settings 页面支持托管加密凭据管理，保存后不再显示或持有明文 Key。
- 聊天链路支持通过 `credential_id` 调用 hosted-provider-chat，前端继续只负责 Context Builder 和 provider messages 构建。
- Data Management 明确不导出托管凭据，也拒绝导入外部 provider credential 数据。

## 加密算法

- 使用 Edge Function 运行时的 WebCrypto `crypto.subtle`
- 算法：`AES-GCM`
- `iv` 每次保存随机生成
- 密文与 `iv` 分开存库
- `key_fingerprint` 使用服务端 HMAC-SHA256 生成，不可反推明文

## 环境变量

- `API_KEY_ENCRYPTION_SECRET`
  - 用于 Edge Functions 加密/解密托管 API Key
  - 只应通过 Supabase Secrets 配置
  - 不能放入任何 `VITE_*` 变量

## 前端设置页变化

- 保存模式新增：
  - `仅本次会话`
  - `当前设备`
  - `托管加密 / 跨设备同步`
- 托管模式支持：
  - 保存托管凭据
  - 测试凭据
  - 设为默认
  - 删除凭据
  - 选择当前使用的凭据
- UI 只展示 metadata，不展示明文 Key
- 保存成功后立刻清空输入框中的明文 Key

## Provider Gateway 集成方式

- `session_only` / `local_device` 保持原有本地 BYOK 路径
- `hosted_encrypted` 通过 `credentialId` 走 Edge Function
- `Context Builder`、`system prompt`、`provider messages` 仍在前端生成
- `hosted-provider-chat` 只负责：
  - 校验用户
  - 解密 Key
  - 代理调用 Provider
  - 返回聊天结果

## Data Management 安全兼容

- 导出默认不包含：
  - `provider_credentials`
  - `provider_credential_secrets`
  - `encrypted_api_key`
  - `encryption_iv`
- 导入时如果检测到 `provider_credentials` 或 `provider_credential_secrets`，会直接拒绝
- 页面安全说明补充：
  - 托管 API Key 不会被导出
  - 恢复备份后需要重新配置 API 凭据

## 安全边界

- 浏览器 localStorage 不保存 hosted API Key 明文
- 前端保存成功后不继续持有 hosted API Key 明文
- `provider_credentials` 不保存明文 Key
- `provider_credential_secrets` 只保存密文和 IV
- 普通前端不能直接 `select provider_credential_secrets`
- Edge Function 响应不返回明文 Key
- 不在 console / 错误信息 / `context_runs` / 备份 JSON 中写入明文 Key
- 删除凭据后，`credential_id` 无法继续用于测试或聊天
- 用户 A 不能测试、调用、删除用户 B 的凭据
- 未登录用户不能调用 hosted credential functions
- Demo 模式继续只走 Mock

## 手动部署步骤

1. 在 Supabase SQL Editor 执行：
   - `supabase/migrations/0008_phase8_1_hosted_encrypted_api_keys.sql`
2. 设置 Supabase Secret：
   - `API_KEY_ENCRYPTION_SECRET`
3. 部署 Edge Functions：
   - `hosted-credentials-save`
   - `hosted-credentials-list`
   - `hosted-credentials-test`
   - `hosted-credentials-set-default`
   - `hosted-credentials-delete`
   - `hosted-provider-chat`

## 测试方法

1. 登录普通用户账号，进入 Settings。
2. 选择 `托管加密 / 跨设备同步`，填写 Provider、Base URL、Model、API Key。
3. 保存后确认：
   - 不再显示明文 Key
   - 只显示 metadata
4. 点击“测试连接”，确认可返回成功或中文错误。
5. 在聊天页使用 hosted 模式发送消息，确认能正常得到回复。
6. 刷新页面或换设备登录，确认默认托管凭据可被重新发现并继续使用。
7. 导出 JSON，确认不包含 provider credentials / secrets。
8. 人工尝试构造包含 provider credentials 的导入 JSON，确认被拒绝。
