# 部署上线指南

本文档说明如何将角色酒馆部署到生产环境。

## 前置条件

- [Supabase](https://supabase.com) 账号和项目
- [Vercel](https://vercel.com) 账号（或其他静态托管平台）
- Node.js 18+ 和 npm

---

## 一、本地启动

```bash
npm install
cp .env.example .env
# 编辑 .env 填入开发环境的 VITE_SUPABASE_URL 和 VITE_SUPABASE_ANON_KEY
npm run dev
```

---

## 二、Supabase 数据库 Migration 执行顺序

在 Supabase Dashboard → SQL Editor 中，按顺序执行以下文件：

```
0001_initial_schema_draft.sql
0002_v41_owner_admin_policy_patch.sql
0003_v42_user_api_credentials.sql
0004_v43_product_completion_tables.sql
0005_auth_user_bootstrap.sql
0006_phase6_query_indexes.sql
0007_phase7_versions_branches_soft_delete.sql
0008_phase8_1_hosted_encrypted_api_keys.sql
```

> 注意：必须严格按顺序执行，每个文件执行后确认无报错再执行下一个。

## 三、Supabase Secrets 设置

在 Supabase Dashboard → Settings → Edge Functions → Secrets 中设置：

| Secret 名称 | 说明 |
|-------------|------|
| `API_KEY_ENCRYPTION_SECRET` | 托管加密主密钥，用于 AES-GCM 加密用户 API Key。建议使用强随机 Base64 字符串，至少 32 字节。**绝对不能暴露到前端或 VITE_ 变量中。** |

生成强随机密钥示例：
```bash
node -e "console.log(require('crypto').randomBytes(32).toString('base64'))"
```

## 四、Edge Functions 部署

```bash
# 安装 Supabase CLI 并登录
npx supabase login

# 链接到你的 Supabase 项目
npx supabase link --project-ref <your-project-ref>

# 部署所有 Edge Functions
npx supabase functions deploy hosted-credentials-save
npx supabase functions deploy hosted-credentials-list
npx supabase functions deploy hosted-credentials-test
npx supabase functions deploy hosted-credentials-set-default
npx supabase functions deploy hosted-credentials-delete
npx supabase functions deploy hosted-provider-chat
```

## 五、Supabase Auth 配置

在 Supabase Dashboard → Authentication → Settings 中：

1. **Site URL**: 填入生产域名，例如 `https://your-app.vercel.app`
2. **Redirect URLs**: 添加以下地址：
   - `https://your-app.vercel.app`
   - `https://your-app.vercel.app/login`
   - `https://your-app.vercel.app/roleplay`
   - 以及本地开发地址 `http://localhost:5173`
3. **Email Auth**: 确认 Email 登录已启用（默认启用）

## 六、Vercel 部署

### 环境变量

在 Vercel Dashboard → Settings → Environment Variables 中设置：

| 变量名 | 值 | 说明 |
|--------|-----|------|
| `VITE_SUPABASE_URL` | `https://xxxxx.supabase.co` | Supabase 项目 URL |
| `VITE_SUPABASE_ANON_KEY` | `eyJhbGciOi...` | Supabase 匿名 Key（公开安全） |
| `VITE_APP_MODE` | `production` | 运行模式 |

> 注意：`API_KEY_ENCRYPTION_SECRET` 只设置在 Supabase Secrets 中，不要放在 Vercel 环境变量。

### 部署步骤

1. 将代码推送到 GitHub 仓库
2. 在 Vercel 中 Import 该仓库
3. 配置：
   - **Build Command**: `npm run build`
   - **Output Directory**: `dist`
   - **Framework**: Vite
4. 添加上述环境变量
5. 部署

### SPA Rewrite + 安全 Headers

推荐 `vercel.json` 配置（根目录）：

```json
{
  "rewrites": [{ "source": "/(.*)", "destination": "/index.html" }],
  "headers": [
    {
      "source": "/(.*)",
      "headers": [
        { "key": "X-Content-Type-Options", "value": "nosniff" },
        { "key": "X-Frame-Options", "value": "DENY" },
        { "key": "Referrer-Policy", "value": "strict-origin-when-cross-origin" },
        { "key": "Permissions-Policy", "value": "camera=(), microphone=(), geolocation=()" }
      ]
    }
  ]
}
```

> CSP 暂不强制，避免破坏 Supabase / API 资源加载。如需 CSP，建议先以 Report-Only 模式测试。

## 七、CORS 配置

Edge Functions 的 CORS 默认允许 `localhost:5173` 和 `127.0.0.1:5173`。

生产环境需要设置 Supabase Edge Function Secrets 中的 `APP_PUBLIC_URL` 或 `ALLOWED_APP_ORIGIN` 为你的生产域名：

```
APP_PUBLIC_URL=https://your-app.vercel.app
```

多个域名可以用逗号分隔设置在 `ALLOWED_APP_ORIGIN` 中。

> 注意：不要在代码中写死生产域名。

## 八、hosted_encrypted 托管加密注意事项

1. **非流式回复**: 托管加密模式通过 Edge Function 代理调用 AI Provider，当前为一次性返回完整回复，不支持逐字流式输出。用户界面会有明确提示。
2. **API Key 安全**: 明文 Key 仅在保存时通过 HTTPS 传输一次，之后所有调用由 Edge Function 解密后代理。
3. **跨设备同步**: 登录同一账号后，默认托管凭据会自动被发现和使用。
4. **备份不含凭据**: 导出备份 JSON 不包含托管凭据的明文或密文。恢复备份后需要重新配置 API。

## 九、数据备份和恢复建议

### 用户自助备份

用户可在"设置 → 数据管理"中导出 JSON 备份，包含：
- 角色、提示词模板、世界书、记忆、会话、消息、版本历史

不包含：
- API Key（BYOK 和托管均不导出）
- context_runs（默认不导出）

### 数据库级备份

建议在 Supabase Dashboard 中定期执行数据库备份：
- Supabase → Database → Backups
- 根据套餐选择自动备份频率

### 恢复注意事项

- 导入的备份数据会作为"副本"导入，不会覆盖现有数据
- 导入后 API 凭据需要重新配置
- 恢复回收站中的项目可通过"数据管理 → 回收站"

## 十、常见问题

### Q: 部署后页面白屏？
A: 检查 Vercel 环境变量是否正确设置了 `VITE_SUPABASE_URL` 和 `VITE_SUPABASE_ANON_KEY`。Vercel 部署时需要重新部署才能使环境变量生效。

### Q: 登录后无法聊天？
A: 确认用户在"设置"中配置了 API Key（本地 BYOK 或托管加密），或检查 Provider 连接测试是否通过。

### Q: Edge Function 调用失败？
A: 检查 Edge Functions 是否已全部部署，以及 `API_KEY_ENCRYPTION_SECRET` 是否在 Supabase Secrets 中正确设置。

### Q: 托管加密模式下回复很慢？
A: 托管模式为非流式回复，需要等待 AI 生成完整回复后才返回，体验上会比流式慢。可切换到"当前设备"模式使用流式回复。

### Q: CORS 报错？
A: 确认 `APP_PUBLIC_URL` 或 `ALLOWED_APP_ORIGIN` 已设置为生产域名，或检查 Edge Functions 是否已重新部署。

### Q: 迁移执行顺序错了怎么办？
A: 如果还未上线，可以在 Supabase Dashboard 中重置数据库后重新按顺序执行。如果已上线，请联系 DBA 评估影响。
