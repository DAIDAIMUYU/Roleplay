# 技术债与重构计划

本文件只做规划，不在当前阶段实现重构。

## 1. ChatRoomPage 拆分计划

- 风险等级：中。
- 推荐时机：移动端和设置页稳定性验收后。
- 建议拆分：
  - 移动端顶部栏
  - 空状态
  - 新建会话 / 角色选择 Modal
  - 桌面三栏布局壳
  - 发送区状态提示
- 注意：不改 `useChatSession` 行为，不改 Context Builder。

## 2. ContextPreview 拆分计划

- 风险等级：中高。
- 推荐时机：先补 E2E 冒烟后。
- 建议拆分：
  - 当前状态
  - 角色设定
  - 模板
  - 世界书
  - 记忆
  - 摘要
  - 用量与费用
  - 高级信息 / System Prompt
- 注意：最终 System Prompt 与真实请求必须保持一致。

## 3. Settings API 模块拆分计划

- 风险等级：中。
- 推荐时机：设置页 API 最终验收后。
- 建议拆分：
  - Provider 预设选择
  - 当前账号云端配置列表
  - 本设备配置折叠区
  - API 编辑 Modal
  - 测试与启用状态
- 注意：不要打破 hosted_encrypted / local_device / session_only 的行为边界。

## 4. Studio Editor 表单抽象计划

- 风险等级：中。
- 推荐时机：移动端长表单验收后。
- 建议抽象：
  - 通用字段组件
  - 通用 sticky footer
  - safe-area padding
  - 错误显示
  - 保存/取消按钮布局

## 5. 错误处理统一

- 风险等级：低到中。
- 推荐时机：可分批推进。
- 目标：
  - 统一 unknown error → 用户可读错误。
  - 避免 `[object Object]`。
  - 不打印 API Key、Authorization header 或 secret。

## 6. 自动化测试

- 风险等级：低。
- 推荐时机：第二阶段 A/B 完成后。
- 最低覆盖：
  - `/` 首页打开
  - `/help` 移动端无横向溢出
  - `/studio` 创建角色 Modal
  - `/roleplay` 新建会话角色选择
  - `/settings` API 配置列表与编辑

## 7. 不建议短期动的模块

| 模块 | 原因 |
| --- | --- |
| Context Builder 核心 | 影响聊天真实请求、Token 预算、世界书、记忆、摘要链路 |
| Provider Gateway | 涉及 session/local/hosted 三条 API 调用路径 |
| 数据库 schema | 已多轮稳定，改动需要 migration 和 RLS 验证 |
| Supabase 同步核心 | 涉及本地/云端一致性和用户数据安全 |

短期应以测试、拆分展示层、文案和体验收口为主。
