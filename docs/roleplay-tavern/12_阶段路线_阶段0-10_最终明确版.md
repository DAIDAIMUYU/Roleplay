# 12_阶段路线：阶段 0-10 最终明确版

> 本文件是 V4.3 的阶段执行主文件。所有阶段必须按顺序推进，不允许跳阶段，不允许用 Mock 冒充真实完成，不允许删除需求文档。

## 阶段成熟度总览

| 阶段范围 | 产品成熟度 | 完成后的状态 |
|---|---|---|
| 阶段 0-4 | MVP 初版 | 可启动、可登录、可配置自己的 API、可真实聊天、Demo 不烧钱 |
| 阶段 5-6 | 角色扮演核心版 | 角色卡、提示词、世界书、记忆、上下文引擎基本完整，适合长期 RP |
| 阶段 7-9 | 产品级完整版 / 核心封版 | 多角色、分支、备份、后台、分享、PWA、全局搜索、标签中心、AI 创作助手、移动端体验完善 |
| 阶段 10 | 可选生态与商业化扩展 | 市场、社区、支付、多模态、公用模型托管等可选增强 |

# 阶段 1：运行模式 + Auth + 数据库 + RLS + Repository

## 目标

确立产品安全边界和数据地基：访客 Demo、登录用户、管理员、站主、数据隔离、数据库结构、Repository 层。

## 必须完成

### 运行模式

- Demo Mode：访客不登录、不写正式数据库、不调用真实 AI。
- Authenticated User Mode：普通用户登录后管理自己的数据。
- Admin Mode：管理员管理 Demo、系统模板、公告、反馈等运营内容。
- Owner Mode：站主最高权限，管理管理员、系统级配置和后台。

### Auth

- Supabase Auth 接入。
- 登录 / 注册基础流程。
- 当前用户状态管理。
- Owner / Admin / User 角色读取。
- Guest 不作为默认 Auth 用户，除非后期明确引入匿名登录。

### 数据库与 RLS

- profiles / user_roles。
- characters / sessions / branches / participants / messages。
- prompt_templates。
- worldbooks / worldbook_entries。
- memories。
- context_runs。
- trash_items / backup_artifacts。
- audit_events。
- model_presets。
- 所有私有表按 `user_id` 隔离。
- RLS 默认拒绝跨用户读取。
- Admin 默认不能读取普通用户 private sessions/messages/memories/context_runs/API credentials。

### Repository

- 所有数据访问必须通过 repository/service。
- UI 组件不得直接散落 Supabase 查询。
- Repository 必须区分 Demo 数据、用户私有数据、系统模板数据。

## 禁止事项

- 不在阶段 1 写真实聊天。
- 不把 API Key 存进数据库。
- 不让 Admin 默认看用户私聊。
- 不让 Demo 写正式业务库。

## 验收标准

- 用户 A/B 的 private 数据互不可见。
- Admin 可以管理 demo/system/admin 内容，但不能默认读取普通用户 private 聊天。
- Owner 可以管理角色权限。
- Repository 单元测试或最小验证通过。
- Git 提交：`phase-1-auth-db-repository`。

# 阶段 2：产品信息架构 + UI App Shell

## 目标

建立产品级 UI 外壳和信息架构，不再做“功能堆叠页面”。

## 必须完成

### 五大产品空间

- Hall 酒馆大厅：访客/用户入口、Demo 入口、最近会话、快速开始。
- Chat Room 聊天房间：聊天主体验。
- Studio 创作工坊：角色、世界书、记忆、提示词模板管理入口。
- Settings 设置中心：账号、API、外观、数据、安全设置。
- Admin 后台：Owner/Admin 管理入口。

### 桌面端 UI

- 顶部导航或侧边导航清晰。
- Chat 页面可三栏，但聊天区必须为核心视觉焦点。
- 右侧面板定位为“上下文控制台”，不是表单堆叠区。
- 参考图只作为风格氛围参考，不要求 100% 复刻。

### 移动端 UI

- 按 App 思维设计，不压缩桌面三栏。
- 底部 Tab：大厅 / 聊天 / 工坊 / 设置 / 我的，可按实际优化。
- Chat 使用底部输入框、Bottom Sheet、移动端安全区。
- 角色卡、世界书、记忆、上下文预览用抽屉或页面切换。

### 状态设计

- Demo 状态。
- 未登录状态。
- 已登录未配置 API 状态。
- API 连接失败状态。
- 空角色 / 空会话 / 空世界书 / 空记忆。
- 加载状态、错误状态、保存状态。

### Onboarding

- Demo 用户：说明“当前为模拟体验，不调用真实模型”。
- 登录用户：引导配置 API、创建角色、创建会话、开始聊天。

## 禁止事项

- 不接真实 LLM。
- 不在 Chat 页面堆完所有管理表单。
- 不牺牲移动端体验去复刻桌面参考图。

## 验收标准

- 访客打开后能明确知道自己处于 Demo 模式。
- 登录用户能看到下一步引导。
- 移动端像 App，不像缩小网页。
- Git 提交：`phase-2-app-shell`。

# 阶段 3：Provider Gateway + Demo Mock + BYOK 本地版

## 目标

解决“谁的 API、怎么调用、怎么不烧站主的钱”。

## 必须完成

### Provider Gateway

- 统一模型调用入口。
- Mock Provider。
- DeepSeek Provider。
- OpenAI-compatible Provider 结构。
- Provider 错误标准化。
- 流式输出接口预留。

### Demo Mock

- 访客 Demo 只能使用 Mock Provider。
- Demo 回复必须明确标识为模拟回复或演示模式。
- Demo 不调用站主 API，不调用用户 API。

### BYOK 本地版

- 登录用户可配置自己的 Provider。
- 支持临时 API Key：仅当前会话保存。
- 支持本设备 API Key：当前浏览器保存，不上传服务器。
- 支持 baseURL、model、temperature、maxTokens、contextLimit。
- 支持连接测试。
- 支持错误翻译：401、402、429、模型不存在、网络失败、超时等。
- UI 明确说明：阶段 3 Key 不跨设备。

## 禁止事项

- 不把用户 API Key 明文存入数据库。
- 不实现服务端加密托管 API Key。
- 不允许访客使用站主 API。
- 不把 service role key 放前端。

## 验收标准

- Demo 聊天不产生真实 API 请求。
- 用户输入自己的 API Key 后连接测试可运行。
- 无 API 时 UI 不崩，显示配置引导。
- Git 提交：`phase-3-provider-byok-local`。

# 阶段 6：世界书 / 记忆 / 上下文引擎 / 快捷指令

## 目标

解决长篇 RP 的稳定性和可解释性。

## 必须完成

### 世界书

- 世界书 CRUD。
- 条目 CRUD。
- 分类：地点、人物、势力、物品、事件、规则、术语。
- 关键词 / 备用关键词。
- 优先级。
- 启用 / 禁用。
- 作用范围：全局、角色、会话、用户设定。
- token 估算。
- 最大注入条数。
- 命中预览。
- 命中解释。
- 世界书搜索。

### 记忆

- 记忆 CRUD。
- 类型：短期、长期、摘要、事件、关系、用户偏好、角色偏好。
- AI 建议记忆 → 用户确认 → 保存。
- 手动编辑 / 删除 / 禁用。
- 重要程度、可信度、来源消息。
- 当前注入记忆预览。
- 记忆搜索。

### 上下文引擎

- Context Builder。
- Context Preview。
- Context Debugger。
- context_runs。
- token 预算。
- 裁剪报告。
- 本轮最终发送内容预览。
- 本轮注入角色、世界书、记忆、消息列表。

### 快捷指令

- `/summary`：生成或更新剧情摘要。
- `/memory`：将内容保存为记忆建议。
- `/scene`：修改当前场景。
- `/ooc`：临时出戏指令。
- 快捷指令在移动端可通过工具菜单选择。

## 禁止事项

- 不允许自动无审核写入长期记忆。
- 不允许无上限注入世界书或记忆。
- 不允许用户不知道 AI 本轮读了什么。

## 验收标准

- 用户能看到本轮命中了哪些世界书。
- 用户能确认或拒绝记忆建议。
- 用户能查看上下文预览和裁剪报告。
- 角色长聊稳定性明显提升。
- Git 提交：`phase-6-worldbook-memory-context`。

# 阶段 10：可选生态与商业化扩展

## 目标

阶段 10 不属于核心产品必须完成项。它用于项目真正做起来后，扩展生态、社区、商业化和多模态能力。

## 可选交付

- 模板市场。
- 角色市场。
- 世界书市场。
- 社区系统。
- 用户公开主页。
- 评论 / 收藏 / 评分。
- 支付系统。
- 订阅系统。
- 公共模型托管。
- 访客有限真实模型体验。
- 语音输入 / 语音回复。
- 图片生成 / 角色头像生成。
- 多模态剧情素材管理。
- 站内通知。
- 推送通知。
- 更完整的管理员分权系统。

## 阶段定位

- 如果目标是“私人/小范围可用的完整产品”，阶段 9 已经足够。

- 阶段 10 不允许破坏阶段 0-9 已经完成的安全、数据隔离和用户隐私边界。

## 验收标准

- 商业化功能和核心 RP 功能隔离。
- 支付、市场、社区不影响私人数据安全。
- 公开内容与 private 数据边界清晰。
