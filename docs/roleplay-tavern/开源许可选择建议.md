# 开源许可选择建议

本仓库当前**尚未确定开源许可证**。在正式确定许可证之前，默认保留所有权利。

如果你打算在本项目基础上二次开发、公开部署或分发，请先选择合适的开源许可证。

## 常见许可证对比

| 许可证 | 商业使用 | 修改后闭源 | 专利授权 | 网络服务条款 |
|--------|:---:|:---:|:---:|------|
| MIT | 是 | 是 | 否 | 无要求 |
| Apache-2.0 | 是 | 是 | 是 | 无要求 |
| AGPL-3.0 | 是 | 否 | 是 | 网络使用也需开源 |
| GPL-3.0 | 是 | 否 | 是 | 分发需开源 |

## 推荐场景

### MIT
- 想最大程度开放，允许任意使用
- 不关心别人闭源商用
- 适合个人项目、工具库

### Apache-2.0
- 类似 MIT，但增加了专利授权保护
- 适合有专利顾虑的项目
- 企业级项目常用

### AGPL-3.0
- 要求任何网络使用（包括 SaaS）也必须开源
- 适合希望保持开放、防止被云服务商"白嫖"的项目
- 对企业用户不友好，可能限制采用

### 暂不开源 / Proprietary
- 目前不想授予他人使用权利
- 可以在未来再选择许可证

## 建议

**如果你是个人开发者，推荐 MIT**：最简单，最宽松，最容易被社区接受。

**如果你想保护贡献者不被专利诉讼威胁，推荐 Apache-2.0**。

**如果你希望保持项目开放性，不想被人拿去做成闭源 SaaS，推荐 AGPL-3.0**。

## 如何添加许可证

1. 选择上述许可证之一
2. 在仓库根目录创建 `LICENSE` 文件
3. 将许可证全文复制进去
4. 更新 `README.md` 中的 License 章节
