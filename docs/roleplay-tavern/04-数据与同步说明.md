# 数据与同步说明

## 1. 本地优先数据模型

Roleplay Tavern 默认本地优先。未登录用户的数据保存在当前浏览器本地数据库中。

## 2. IndexedDB 与 localStorage 职责

- IndexedDB：角色、会话、消息、世界书、记忆、本地镜像等结构化数据。
- localStorage：本设备 API 配置、少量 UI 状态和本地偏好。

`local_device` API Key 不跨设备，也不进入云端同步。

## 3. 登录后的云端同步

登录后可以使用云端数据库保存角色、会话、世界书、记忆等数据。系统不应在用户未确认的情况下静默上传本地数据。

## 4. 本地镜像

云端模式下，当前设备也保留本地镜像，用于提升访问速度和降低云端异常时的体验风险。

## 5. 导入导出

数据管理支持导出 JSON 备份和导入恢复。备份文件默认不包含 API Key、密文、IV 或 Secrets。

## 6. 备份与恢复

建议长期使用时定期导出备份。导入时应先预览数据数量，再确认写入。

## 7. 冲突处理现状

当前同步策略偏保守，避免静默覆盖。更完整的冲突解决 UI 列入第二阶段计划。

## 8. 风险说明

- 清除浏览器网站数据会删除本地数据。
- 无痕模式关闭后本地数据可能丢失。
- 更换浏览器或设备不会自动带走本地数据。
- 退出登录不应删除当前浏览器本地数据。
