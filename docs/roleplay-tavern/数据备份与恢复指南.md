# 数据备份与恢复指南

## 为什么需要备份

浏览器本地数据（IndexedDB）在以下情况下可能**永久丢失**：
- 清除浏览器网站数据
- 使用无痕/隐私模式并关闭窗口
- 更换浏览器
- 更换设备
- 格式化或重置设备
- 浏览器数据损坏

云端同步**不是**备份的替代品。云端同步帮助你多设备访问，但如果误操作删除了数据，同步可能会传播删除。定期导出 JSON 备份是最可靠的数据保护方式。

## 如何导出备份

1. 打开设置 → 数据管理
2. 选择要导出的数据类型
3. 点击「导出备份」
4. 保存下载的 JSON 文件到安全位置（如云盘、外置硬盘）

## 导出包含什么

| 数据类型 | 是否导出 |
|---------|---------|
| 角色卡 | 是 |
| 提示词模板 | 是 |
| 世界书与词条 | 是 |
| 记忆 | 是 |
| 会话 | 是 |
| 消息 | 是 |
| 消息版本历史 | 是 |
| API Key | **否** |
| 托管凭据密钥 | **否** |
| encrypted_api_key | **否** |
| encryption_iv | **否** |

## 如何导入恢复

1. 打开设置 → 数据管理
2. 选择之前导出的 JSON 备份文件
3. 预览导入内容
4. 选择导入模式（复制为新记录 / 覆盖现有）
5. 点击「导入」

导入时会保留原始数据 ID 和时间戳。如果存在冲突（同 ID 不同内容），会生成副本而非覆盖。

## 备份频率建议

- **频繁使用者**：每周导出一次
- **偶尔使用者**：每月导出一次，或在重要创作后导出
- **长期不使用者**：在清浏览器数据或换设备前导出

## 备份文件安全

- 导出的 JSON 文件**不含 API Key**，但包含你的角色、对话等创作内容。
- 请将备份文件保存在安全位置，不要公开分享。
- 如果需要导入到新设备，你需要在设置中重新配置 API Key。

## 云端同步 vs 备份

| | 云端同步 | 导出备份 |
|------|---------|---------|
| 用途 | 多设备访问 | 灾难恢复 |
| 覆盖方式 | 手动选择上传/下载 | 手动导入 |
| 包含 API Key | 否 | 否 |
| 离线可用 | 否 | 是 |
| 版本历史 | 否 | 取决于备份文件 |

建议：两者结合使用。云端同步用于日常多设备访问，定期导出备份用于安全底线。
